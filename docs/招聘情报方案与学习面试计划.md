## 一、招聘情报采集方案（先讲现实约束）

### 1.1 现实约束（已实测）
- **BOSS 直聘无法自动爬**：实测你给的急招深链 `m.zhipin.com/.../weijd-job/...` 返回的是「打开 APP / 登录墙」，
  移动深链带 `sid` 会话参数，无登录态拿不到 JD 正文；BOSS 反爬（设备指纹 + 行为风控）也禁止脚本化抓取。
- **猎聘 / 智联等同理**：列表页强 JS 渲染 + 登录态，纯 httpx 抓取收益极低、且易触发封禁。
- **官网招聘页（careers.xxx.com）相对友好**：多为静态或服务端渲染，是「自动可探」的最佳源；
  但岗位详情仍需人工复核（薪资 / 急招标记常常只在列表态可见）。

### 1.2 推荐架构：情报工作流（而非爬虫）
```
官方招聘源(自动探/手动核对)  ┐
BOSS/猎聘(用户转发 JD 文本)   ├─► 结构化入库(JD 情报表) ─► 招聘情报台(前端)
官网 JD 链接(粘贴即解析)     ┘                                  │
                                                         急招/社招 · 梯队/方向筛选 · 学习/面试联动
```

**三句话原则**
1. **官方源为主、第三方平台为辅**：官方站能自动探就自动探，BOSS 只做「用户粘贴 → 结构化」的入口。
2. **人工粘贴即结构化**：你只要把 BOSS / 官网 JD 的文字或链接丢进来，后端解析成
   `公司 / 岗位 / 薪资 / 城市 / 急招or社招 / 方向 / 来源 / 日期 / 备注`，进库即上墙。
3. **和我们 20 厂 / 方向强绑定**：每条情报自动按 `tier + direction` 归类，和现有公司卡、JD 画像联动。

## 三、学习项目计划（Phase A→G 贯穿作品集）

> 现有数据里每家公司都带 `phaseFocus`（如腾讯 `C→B→G→D`、Kimi `F→G→E→C`）。
> 学习计划就是把所有目标厂的 phase 合并成一条**递进路线图**，用一个作品集 Agent 项目全部覆盖。

### 3.1 Phase 字典（全站通用）
- **A** 工具网关 / Tool Learning / Function Calling / MCP
- **B** 记忆 / 多轮对话
- **C** RAG 防幻觉 / 事实核查 / 搜索式 RAG
- **D** 多 Agent 编排 / 反思防环（LangGraph）
- **E** 评测闭环（SWE-Bench / TAU-Bench 同构）
- **F** 推理优化（vLLM / 量化 / continuous batching）
- **G** 多厂商注册表 / 结构化输出 / 混元优先

### 3.2 八周路线图（每个阶段都对应能打动的具体厂）
| 周 | 阶段 | 交付物 | 主要打动 |
|----|------|--------|----------|
| W1–2 | A + G | FastAPI + 混元/千问 API + MCP 工具网关（天气/搜索/DB） | 腾讯/阿里/百度 |
| W3 | B + C | 多轮记忆 + RAG（TF-IDF+2-gram+RRF），带 citation / hallucination 指标 | 字节/腾讯/智谱 |
| W4 | D | LangGraph 多 Agent 编排 + 反思节点防环 | 字节/智谱/快手 |
| W5 | E | 自建 bench，量化 citation_rate / hallucination_rate | 字节/Kimi/阶跃 |
| W6 | F | vLLM 部署 + 量化 + 压测，报 speedup≈8x | Kimi/DeepSeek |
| W7 | 具身扩展 | 机器人任务规划 / 端侧量化蒸馏原型 | 宇树/智元/银河 |
| W8 | 叙事 | 简历 hook 对齐 + 模拟面试 + 作品集上线 | 全部 |

**核心作品集指标（简历固定数字资产）**：`citation_rate=1.0`、`hallucination_rate=0.0`、
`kv_cache_hit_rate≈0.8`、`speedup≈8x`。每版简历复用。

## 四、面试准备框架

### 4.1 按角色簇拆题
1. **Agent 应用开发**（字节Seed / 腾讯混元 / 智谱 / 快手 / 阶跃 / 百度 / 美团）
   - 系统设计：设计一个 RAG + Multi-Agent 系统（画数据流 + 失败模式）
   - 必考：RAG 防幻觉、Function Calling、LangGraph 编排、评测指标定义
   - 作品集映射：W3–W5 的 Agent 项目
2. **推理 / 系统**（Kimi / Kimi Code / DeepSeek）
   - 必考：vLLM 架构、continuous batching、量化、KV cache、分布式训练、开源贡献
   - 作品集映射：W6 推理优化 + 两个 Trending 项目复用同一 core
3. **AI PM**（京东 / 百度 AppBuilder）
   - 必考：低代码生态、竞品分析、AI 链路设计、指标定义（留存 / 采纳率）
4. **具身 Agent**（宇树 / 智元 / 银河 / 星海图 / 星动 / 灵初）
   - 必考：VLA、机器人任务规划、Sim2Real、端侧量化、数据飞轮

### 4.2 各厂简历 hook（已存在于 `seed_data.py` 的 `jd.hook`）
- 字节：Phase D 编排 + Phase E 评测 + Phase C 搜索 RAG
- 腾讯：Phase C 防幻觉 + Phase B 记忆 + Phase G 混元优先 + Phase D 反思强化
- 智谱：Phase D MultiAgent + Phase A 工具网关 + Phase B 长文本 + Phase E 评测
- Kimi：Phase F 推理优化 + Phase G 结构化输出 + Phase E 评测 + 开源
- DeepSeek：系统/可观测 → F 极致优化 → 开源复用 → 评测
- 具身三厂：任务规划 → 端侧量化蒸馏 → 工具调用 → 反思

### 4.3 通用答题框架（STAR + 数字）
- 每道题用「情境 → 任务 → 行动 → 结果（带上面 4 个固定数字）」收口
- 把作品集项目当成万能案例库，按厂 hook 切换侧重点
